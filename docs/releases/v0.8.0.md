## Sprint Summary

Four-pass code-quality scrub on the v0.7.0 tree as the reference implementation for OWB Sprint 31. Each pass ran as its own sub-branch + PR back to the sprint integration branch:

1. **HMB-S033 — `/simplify`** (PR #2): clarity + reuse, no deletions. Two safe refactors landed (envrc dedupe, redundant param drop). Most reviewer findings rejected after verification.
2. **HMB-S034 — `/code-review`** (PR #3): security + correctness. 12 findings fixed across redaction, subprocess hardening, and atomic write. 3 backlog stories filed (HMB-S039/S040/S041) for findings deferred per AC-2.
3. **HMB-S035 — `/refactor-clean`** (PR #4): dead-code removal. Two internal deletions + one public API removal. Established `docs/public-api-changes.md` as the canonical changelog file for OWB-S134 and bridge consumers to read.
4. **HMB-S036 — pyright strict triage spike** (PR #5): basic 25 / strict 1161 inventories captured. DRN-077 accepts pyright basic mode per DRN-073. Two real bugs surfaced and fixed (urllib.error import, secrets.set narrowing).

## Code Changes

### Added

- `docs/public-api-changes.md` — canonical public-API changelog (HMB-S035)
- `pyrightconfig.json` — basic-mode pyright config per DRN-077 (HMB-S036)
- `docs/pyright-basic-inventory.json` + `docs/pyright-strict-inventory.json` — spike snapshots
- `tests/test_redaction.py` — 14 tests for the extended redaction patterns

### Changed (security)

- `bitwarden.unlock_command` parsed via `shlex.split()` with `shell=False`
- `rotate-key` atomic `.sops.yaml` write via tempfile + `os.replace`
- `rotate-key` and `init` honor `HIMITSUBAKO_SOPS_BIN`
- 30s subprocess timeouts on `rotate-key`, `init` (sops + age)
- SOPS stderr redacted in `SopsBackend._decrypt` / `_encrypt`, `rotate-key`
- Google OAuth `error_description` redacted in `BackendError.detail`
- `_default_post` reports payload byte count only on non-JSON response
- `_redaction.py` extended to match Google refresh tokens (`1//...`) and age private keys (`AGE-SECRET-KEY-1...`)

### Changed (UX / cleanup)

- `rotate-key --new-key` exit code on missing file: 1 → 2 (click parse error)
- `_read_rotation_value` strips `\r\n` (not just `\n`)
- `direnv.generate_envrc()` delegates to `_managed_block()` (dedupe)
- `_rotate_google_oauth` no longer takes redundant `backend_name` param

### Removed

- `GoogleOAuthBackend.credential_name` property — public API removal, documented in `docs/public-api-changes.md`
- `cli.init._build_envrc` — internal v0.1.0 forwarder, no callers
- `HimitsubakoSettingsSource.prepare_field_value` — no-op override

### Fixed

- `google_oauth_rotate.py` — explicit `import urllib.error`
- `cli/secrets.py` — `set_secret` narrowing assert
