# Release Manifest — v0.1.0

**Date:** 2026-04-11
**Tag:** `v0.1.0` (commit `8240d23`)
**Theme:** Foundation + SOPS backend (rescoped SOPS track)

## Overview

First rescoped release of himitsubako. v0.1.0 was originally planned as the full multi-backend cut (SOPS + Keychain + Bitwarden CLI + env + direnv + pydantic-settings + docs site + PyPI publish). After Sprint 1 shipped the SOPS track end-to-end as a standalone usable library, v0.1.0 was rescoped down to match what had actually shipped.

This is **not on PyPI**. The public PyPI release target is v0.3.0 (see HMB-S016).

## Scope Decision

The original 13-story `HIM-S###` plan drafted 2026-04-09 was never executed. Sprint 1 ran a different 16-story `HMB-S###` plan and delivered 6 of those stories. Rescoping v0.1.0 to those 6 stories was the only coherent way to reconcile the design docs with shipped code. Env / Keychain / Bitwarden / direnv / pydantic-settings now land in v0.2.0. CI / docs site / PyPI publication land in v0.3.0. A new v0.1.1 hardening release covers known security gaps in the v0.1.0 surface.

## Known Limitations (v0.1.1 hardening backlog)

1. **T-018** — `hmb get` prints full plaintext to stdout with no `--reveal` gate. Top v0.1.1 priority. See ADR OQ-4 for the UX trade-off.
2. **T-010** — `.secrets.enc.yaml` written with default umask (typically 0644) instead of an explicit 0600 chmod.
3. **T-004** — `SopsBackend` does not set a subprocess timeout on `sops` calls; a hung binary hangs the CLI indefinitely.
4. **T-001** — `SopsBackend` resolves `sops` via PATH only with no absolute-path override config option.
